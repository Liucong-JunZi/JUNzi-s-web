## Bug Report - 2026-03-27 (Round 2)

### 严重 (Critical)

#### Bug #1: OAuth 登录流前后端完全不兼容

**问题描述**: 登录流是两套互相冲突的方案，无法闭环

**前端期望**:
- 点击登录 → 跳转到 `/api/auth/github`
- 回调时调用 `authAPI.callback(code)` → 请求 `GET /api/auth/callback?code=xxx`
- 期望返回 `{ user, token }`，token 存入 sessionStorage

**后端实际**:
- `GET /api/auth/github` → 重定向到 GitHub
- GitHub 回调 `GET /api/auth/github/callback?code=xxx&state=xxx`
- 验证后设置 HttpOnly Cookie，重定向到 `{frontendURL}/auth/callback`
- 不带 code 参数

**结果**: AuthCallback.tsx:12 报 "No authorization code received"
**额外问题**: axios 没有开启 withCredentials，跨端口请求不会带 Cookie；路由守卫只看 Zustand 状态，刷新后被踢回登录页

**参考文件**:
- `frontend/src/api/index.ts:6` - axios 配置
- `frontend/src/api/index.ts:75` - callback 函数
- `frontend/src/views/AuthCallback.tsx:14` - 报错点
- `frontend/src/store/authStore.ts:36` - 状态管理
- `frontend/src/components/ProtectedRoute.tsx:8` - 路由守卫
- `backend/internal/controllers/auth_controller.go:180` - GitHub 回调
- `backend/internal/controllers/auth_controller.go:185` - 重定向
- `backend/internal/routes/routes.go:75` - 路由定义
